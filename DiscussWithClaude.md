## 避坑指南

**最常见的坑：**

1. **上下文丢失** — AI没有记忆，每次新对话都要重新提供背景。解决：把关键信息写进 `CLAUDE.md`，用Claude Code时它会自动加载

2. **任务粒度太大** — 让AI"做整个用户系统"必然失控。解决：TASKS.md里每个任务控制在**1-2小时能完成**的粒度

3. **跳过文档直接编码** — 诱惑很大，后果很惨。解决：强迫自己先产出PRD和ARCH，哪怕只有半页纸

4. **不做增量验证** — 写了500行再测。解决：每个任务完成后立刻运行测试，保持绿色

5. **让AI自由发挥技术选型** — 它会选它"觉得好"的，不一定适合你。解决：在CLAUDE.md里明确技术约束

## 四、完整协作流程（含启动方式和交接）

参考方案最核心的实操机制：**启动Claude Code后，直接粘贴对应角色的指令，Agent自己知道去读哪些文件、做什么、完成后更新什么**。

```
【第0步】项目初始化（开发负责人，一次性）
  创建目录结构
  填写CLAUDE.md（技术栈、git账号、规范）
  初始化PROJECT_STATUS.md、TECH_STATUS.md（空模板）
  git init，团队成员各自配置git账号

【第1步】需求阶段
  产品经理手写 INIT_REQ.md（原始需求，不超过1页，不需要完美）
  → 打开Claude Code
  → 粘贴 AGENT_PROMPTS.md 中的【BA启动指令】
  → BA Agent自动：读INIT_REQ.md → 对话澄清 → 产出PRD.md（含验收标准）
                              → 产出HTML原型 → 更新PROJECT_STATUS.md
  → git commit（产品经理账号）

【第2步】架构阶段
  → 打开Claude Code
  → 粘贴【Arch启动指令】
  → Arch Agent自动：读PRD.md + 原型 → 产出ARCH.md + TASKS.md → 更新PROJECT_STATUS.md
  → git commit（开发负责人账号）
  → 打开Claude Code
  → 粘贴【TechLead启动指令】+ "请审查架构设计"
  → TechLead审查ARCH.md → 写入TECH_STATUS.md
  → 开发负责人决策：通过则进入第3步，需修改则Arch修改

【第3步】开发循环（每个任务）
  
  3a 开工
  → 打开Claude Code
  → 粘贴【Dev启动指令】
  → Dev Agent自动读：PROJECT_STATUS.md（今天做什么）
                      TASKS.md（任务详情）
                      TECH_STATUS.md（有无待处理TechLead要求）
                      PRD.md对应章节 + 对应HTML原型
  
  3b TDD编码
  → 先写单元测试（来自PRD验收标准）
  → 跑测试（红）→ 写实现 → 跑测试（绿）
  → 更新TASKS.md任务状态 → 更新PROJECT_STATUS.md
  → git commit（开发者账号）
    格式："feat(user-service): 实现用户登录 [TASK-003] 测试5/5通过"
  
  3c 代码Review
  → 打开Claude Code
  → 粘贴【Review启动指令】+ 指定任务编号
  → Review Agent读代码 → 写审查意见到TECH_STATUS.md
    格式：R-001 🔴必须修复 / 🟡建议 / 🟢好写法
  → 更新TASKS.md状态为"修复中"
  
  3d 修复
  → Dev Agent读TECH_STATUS.md待处理条目
  → 修复 → 重跑测试 → 绿
  → TECH_STATUS.md标记已解决
  → git commit："fix(user-service): 修复R-001 空指针问题"
  → TASKS.md状态改为"完成"

【第4步】阶段性TechLead审查（每个微服务完成后）
  → 粘贴【TechLead启动指令】+ "请审查user-service代码"
  → TechLead读代码 + TASKS.md + 测试结果
  → 写审查意见到TECH_STATUS.md
  → 开发负责人决策并推进

【第5步】集成验收
  → 本地所有单元测试通过
  → 打包，拷贝到内网
  → 产品经理对照PRD.md验收标准 + HTML原型做端到端测试
  → 更新PROJECT_STATUS.md为完成
  → git tag v1.0
```

## 关于第1点：TechLead合并Review

```
你现在是TechLead角色，读取config/目录，
```
