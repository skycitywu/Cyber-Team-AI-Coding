## 使用说明

### 文件结构
- 每次 techlead 的审查意见都是一个独立的块
- 块之间用 `---` 分隔符明确分开
- 每个审查块包含时间戳和审查编号
- PM 和其他 Agent 可以引用"techlead 最新的意见"

### 审查编号规则
- 从 #1 开始递增
- 格式：【techlead 审查 #N】[审查类型] - [时间戳]
- 审查类型：架构审查 / 代码审查 / 测试审查 / 其他

### 时间戳格式
- 格式：YYYY-MM-DD HH:MM:SS
- 例如：2026-02-19 14:30:45
